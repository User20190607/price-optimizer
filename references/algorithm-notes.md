# 单价调整优化器 —— 算法说明与数学推导

## 1. 核心数学约束：GCD 格点定理

将所有单价统一用"分"（整数）表示后，问题变为：

> 给定整数数量向量 **q** 和单价调整向量 **δ**（分），使 Σ qᵢ·δᵢ = gap

其中 gap = target_cents - orig_total_cents。

**定理（裴蜀定理推广）**：上式有整数解 ⟺ gcd(q₁,…,qₙ) | gap

推论：
- 精确可达的最小步长 = g = gcd(q₁,…,qₙ) 分 = g/100 元
- 若目标不在格点，数学上不可消误差最大为 g/200 元
- 实际操作：`snapped = round(target_cents/g)*g`，误差 = |target_cents - snapped|/100

**示例（电缆报价单）**：
数量为 50, 800, 800, 1000
g = gcd(50,800,800,1000) = 50 分 = 0.50 元
→ 总价只能以 0.5 元为步长变动

## 2. 模式A：等比均匀下浮 + 贪心残差

**目标**：所有单价尽量以相同比例缩放，差异最小化。

**算法**：
1. 计算下浮系数 r = snapped_target / orig_total_cents
2. 每行理论新单价（分）：base[i] = round(orig_cents[i] * r)
3. 计算残差：residual = snapped_target - Σ q[i]·base[i]
4. 贪心分配：按数量从大到小遍历，取 step = floor(residual / q[i])，压入该行

**为什么优先压大数量行**：
大数量行的单价变动 1 分 = 总价变动 q[i] 分，撬动比最高，
同时该行单价的相对畸变 = 1/orig_cents[i] 最小（因为通常大数量行单价也不会过低）。

**复杂度**：O(n log n)，无 MILP 求解器依赖

## 4. 模式C：单一折扣率 + 平衡行

**场景**：审计时需要全程可解释，每行都按同一折扣，差额由一行专用"尾差"行承担。

**算法**：
1. rate = round(snapped_target / orig_total_cents, 6)（保留6位，对应0.0001%精度）
2. 每行新单价 = round(orig_cents[i] * rate) / 100
3. 尾差 = snapped_target - 实际小计  （必须是整数分，因为已做 GCD 吸附）
4. 追加一行"调整尾差项"，数量=1，单价=尾差/100 元

**优势**：
- 所有行折扣率完全一致，审计可验算
- 全程只涉及一个折扣系数，文档中只需注明一个数字
- 尾差行金额通常 < g/100 元，视觉上极小

## 7. 浮点安全说明

所有单价转换到"分"时使用 `Decimal(str(value)) * 100`，
避免浮点二进制误差（如 `4.21 * 100 = 420.99999...`）。
对外输出时 `/100` 还原为元，显示精度 2 位小数。
